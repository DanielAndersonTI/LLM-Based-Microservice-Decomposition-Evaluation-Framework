# PetClinic Pilot Study – Structural Adherence of LLM-Generated Microservice Decompositions

## Objective

This repository contains the artifacts and analysis pipeline used in a pilot study evaluating the structural adherence of microservice decompositions generated by **OpenAI o3 (GPT-4 o3)** for the **Spring PetClinic** monolithic system.

The study compares **zero-shot** and **few-shot** prompting strategies by measuring how well the generated service boundaries align with the actual dependency structure extracted from the source code through static analysis.

---

## Key Metrics

### TVD — Dependency Violation Rate

Measures the proportion of dependencies that cross service boundaries.

```text
TVD = Violations / Total Dependencies
```

Lower values indicate better structural adherence.

### TPD — Dependency Preservation Rate

Measures the proportion of dependencies preserved within service boundaries.

```text
TPD = 1 − TVD
```

Higher values indicate better structural adherence.

### Granularity

The total number of services proposed by the decomposition.

---

## Pilot Results

| Metric | Zero-shot | Few-shot |
|----------|----------|----------|
| Structural Violations | 9 | 8 |
| TVD | 22.5% | 20.0% |
| TPD | 77.5% | 80.0% |
| Granularity | 8 | 8 |

## Reproducing the Pilot Study

### 1. Clone Spring PetClinic

Place the original Spring PetClinic source code inside `input/monolith/`:

```bash
cd input/monolith

git clone https://github.com/spring-projects/spring-petclinic.git spring-petclinic-main
```

Expected structure:

```text
input/
└── monolith/
    └── spring-petclinic-main/
```

### 2. Run Static Analysis

```bash
python pipeline/petclinic_static_analysis.py
```

This script:

- Parses the Java source code;
- Extracts classes and structural dependencies;
- Builds the dependency graph.

### 3. Run Graph Audit

```bash
python pipeline/graph_audit.py
```

This script:

- Validates graph integrity;
- Produces graph statistics;
- Generates an audit report.

### 4. Run Structural Violation Analysis

```bash
python pipeline/violation_detection.py
```

This script:

- Loads the LLM-generated decompositions from the CSV files;
- Maps classes to services;
- Detects dependencies crossing service boundaries;
- Computes TVD, TPD, and Granularity;
- Generates the final analysis reports.

### 5. Inspect Results

Results are available in:

```text
results/
```

## How to Adapt to Other Java Systems

To analyze another Java monolithic system:

1. Replace the source code located in `input/monolith/`.
2. Update the project root constant (e.g., `PETCLINIC_ROOT`) in the analysis scripts.
3. Provide new class-to-service mapping CSV files.
4. Execute the pipeline again.

For non-Java systems, replace the Tree-sitter grammar and adapt the dependency extraction logic accordingly.
